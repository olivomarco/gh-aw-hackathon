# MC Kickoff Script

**Timing:** 09:00–09:30 (15 min welcome + 15 min squad formation)

## 09:00–09:15: Welcome & Format Briefing

[APPLAUSE CUE]

**Good morning, everyone! Welcome to the GitHub Agentic Workflows Hackathon!**

[PAUSE]

I'm thrilled to see all of you here. Today is about building the future—specifically, learning how to craft powerful AI agents using nothing but Markdown and GitHub Actions. No complicated code frameworks. Just clear intent, smart composition, and the power of natural language workflows.

[SHOW SLIDE: Theme — "Building AI Agents in Plain English"]

### What Are GitHub Agentic Workflows?

Imagine telling GitHub "I want a bot that summarizes issues every morning, or automatically triages PRs, or coordinates complex multi-step operations across repos." With GitHub Agentic Workflows, you write that intent in Markdown—just a structured document describing what you want—and GitHub Actions executes it. It's like giving GitHub a job description instead of low-level instructions.

The magic is in the composition: you chain lightweight, reusable workflow blocks together. Add a trigger (an event, a schedule, a command), define the engine (what the AI should do), gate it with safety checks, and you're done. No npm packages. No DevOps wrestling. Just clarity, intention, and automation.

[PAUSE]

**Here's what makes this powerful:**
- **Write in English.** Your workflow is a document people can read, understand, and improve.
- **Run on GitHub.** Triggers are GitHub events—PRs, issues, schedules, manual commands.
- **Safe by default.** We'll show you how to gate outputs, scope permissions, and fail gracefully.
- **Learn together.** Coaches are here to guide, not solve. You'll debug, discover, and grow.

[PAUSE]

### The WTH Format

Today we're running a "What The Hack"—a challenge-based sprint where squads work together to solve real problems.

**Three difficulty tracks:**
- **Track 1: Hello, Agent** — Beginner. Learn the fundamentals: simple workflows, triggers, outputs.
- **Track 2: Repo Concierge** — Intermediate. Automation at scale: multi-repo coordination, smart routing, state management.
- **Track 3: Continuous Intelligence** — Advanced. Complex systems: feedback loops, cross-service orchestration, adaptive behavior.

Each track has challenges you complete in order. There's no race—this isn't about speed. **It's about learning and completion.** We judge on how well your workflow works, how creatively you solved the problem, and how responsibly you built it.

**Your squad size:** 3 to 5 people. Self-organize with folks you want to work with, or we'll help you find a team.

**Your coaches:** [SHOW COACHES] We have [COACH NAME, ROLE], [COACH NAME, ROLE], and [COACH NAME, ROLE] here today. They won't write code for you—they'll ask questions, help you debug, and point you toward resources. Think of them as your guides, not your answer key.

[PAUSE]

### Judging Criteria (The Honest Version)

At the end of the day, we'll look at:
1. **Does it work?** Does your workflow meet the challenge's success criteria?
2. **Is it creative?** Did you think outside the box? Combine features in novel ways?
3. **Is it safe?** Did you use guardrails, scope permissions properly, handle errors?
4. **Is it clean?** Is the Markdown readable? Would someone else understand it?
5. **Is it documented?** Can a stranger run it cold?

**Most importantly: if you complete at least one challenge, you get a Completion Award.** You're not here just to compete—you're here to build and learn. Even if you don't win the main competition, you'll walk away with something real.

[PAUSE]

### How to Get Help

You'll get stuck. That's expected—it's part of the process. **Here's where to go:**

- **Discord #hackathon-help** — Real-time questions. Coaches and team monitor this all day.
- **GitHub Discussions** — Async questions. Great for detailed debugging.
- **Raise your hand.** If Discord isn't working, find a coach in person. We're here.
- **The coaches rotate through squads.** Don't wait—speak up when you hit a wall.

There's a 15-minute rule: if you're stuck, try debugging on your own, then ask. Coaches love helping squads that are actively thinking.

[PAUSE]

## 09:15–09:30: Squad Formation

[SHOW SLIDE: Squad Formation Instructions]

**All right, it's time to form your squads.**

If you came with a team, you're set—grab a table together. If you're new here or flying solo, look around. We'll help matchmake in the next 2 minutes. Find 2–4 people you'd like to work with. Maybe grab someone from a different track so you learn from each other.

[PAUSE for squad self-organization — ~2 minutes]

**Perfect! Here's what's next:**

1. **Pick a squad name.** Make it fun, memorable, whatever reflects your team. You'll be introducing yourselves with this name.
2. **Pick your lead for today.** One person who'll be the main contact for coaches and submissions. Usually this is whoever's most organized, but you decide.
3. **Tell a coach your name.** We need to know who's in each squad so we can check on you.

[PAUSE while squads organize — ~2 minutes]

### Open a Codespace NOW

Here's the most important thing I'm going to say all day: **Open a GitHub Codespace right now. Don't wait. Do it while I'm talking.**

Go to the repo, click the Code button, select Codespaces, and spin one up. Let it boot while we're settling in. The environment is pre-configured—all the tools you need are there. If you wait until you're ready to code, you'll lose 5–10 minutes to setup. Not today.

[PAUSE]

Coaches: once squads are set, walk them through the Codespace setup if anyone's stuck. It's quick—just a couple of clicks.
