# File: engine-swap-copilot.md

---
on:
  schedule:
    - cron: "0 9 * * 1-5"  # Weekday mornings
  workflow_dispatch:

permissions:
  contents: read
  issues: write

safe-outputs:
  create-issue:
    max: 1
    close-older-issues: true
    labels: [dependency-drift, automated]
    title-prefix: "[dep-drift] "
    expires: 3d
  noop:
    reason: "No dependency changes detected in the past 7 days."

# ENGINE: GitHub Copilot
# Best for: quick classification, low cost, tightly structured prompts
# Trade-off: may miss subtle reasoning about dependency implications
engine: copilot

tools:
  bash:
    allow:
      - "git log:*"
      - "git diff:*"
      - "cat:package.json"
      - "cat:go.mod"
      - "cat:requirements.txt"
      - "grep:*"
---

## Goal

Detect dependency drift: identify packages that were added, removed, or
version-bumped in the past 7 days, assess whether any change introduces
security or compatibility risk, and create an issue if action is needed.

## Steps

1. **Collect dependency file changes** using bash:
   ```
   git log --oneline --since="7 days ago" -- package.json go.mod requirements.txt
   git diff HEAD~7 HEAD -- package.json go.mod requirements.txt
   ```

2. **If no dependency files changed**, call `noop` and stop.

3. **Classify each changed dependency:**

   | Change type | Risk level |
   |-------------|-----------|
   | New dependency added | Medium — review for necessity |
   | Dependency removed | Low — verify nothing breaks |
   | Patch version bump (x.y.Z) | Low |
   | Minor version bump (x.Y.z) | Medium — check changelog |
   | Major version bump (X.y.z) | High — breaking changes likely |
   | Known CVE in new version | Critical |

4. **If all changes are low-risk patch bumps**, call `noop`.

5. **Otherwise**, create an issue summarising the changes.

## Output format

```markdown
## 📦 Dependency Drift Report

**Period:** Last 7 days
**Engine:** Copilot

### Changes detected

| Package | From | To | Risk | Notes |
|---------|------|----|------|-------|
{one row per changed dependency}

### Recommended actions

{Bullets for any Medium/High/Critical items. "None" if all Low.}

_Generated by Engine Swap (Copilot version)._
```

   grep -r "require('{package}')" src/ --include="*.js"

## Coaching note — comparing engine outputs

Participants running all three versions on the same commit should observe:
- **Copilot**: fastest, most concise, follows the table structure closely
- **Claude**: richer reasoning paragraphs, more nuanced risk assessment
- **Codex**: deepest code-level analysis, specific API impact, more brittle on prose
