# Challenge 00: Setup & Hello, Agent

**Track:** Onboarding (all squads start here)  
**Difficulty:** 🟢 Beginner  
**Estimated time:** 30–45 minutes  
**Prerequisites:** GitHub account with Codespaces access (or local Docker + VS Code)

---

## Background

**gh-aw** is a framework that lets you write agentic workflows in plain markdown. Instead of YAML, you write your trigger conditions, permissions, and AI instructions as a markdown file with a small frontmatter section. The `gh aw` CLI then compiles your workflow into a GitHub Action (`.lock.yml`) that runs on the GitHub platform, executing your AI-powered logic with safe guardrails.

Today, you'll go from zero to "your first workflow that does something" — creating an issue on your own GitHub repo using an AI agent. This is the smoke test that proves your environment works, builds your confidence, and sets you up to tackle harder challenges.

**Why this matters:** Every participant needs to complete this challenge first. It's not about the complexity—it's about establishing a fast feedback loop. Write → Compile → Run → See result. Once you feel that loop, everything else clicks into place.

---

## Goals

By the end of this challenge, your squad will:

1. ✅ Have a working gh-aw development environment (Codespaces or local Docker + VS Code)
2. ✅ Have authored, compiled, and successfully run your first agentic workflow
3. ✅ Understand the gh-aw frontmatter basics (`on`, `permissions`, `safe-outputs`, `engines`)
4. ✅ Have seen at least one `safe-outputs` action result — a GitHub issue created by your workflow

---

## Challenge

Create a gh-aw workflow file named `hello-agent.md` in `.github/workflows/` that:

- **Triggers manually** via `workflow_dispatch` (no need to wait for pushes or schedules today)
- **Uses an AI engine** (Copilot, Claude, Codex, or Gemini — your choice)
- **Creates a GitHub issue** when run, with:
  - A friendly greeting from your squad
  - One fun fact about agentic workflows
  - A title prefixed with `[hello-agent]`
- **Uses `safe-outputs`** to create the issue (NOT direct GitHub API calls)
- **Has minimal permissions** — read-only access to the repo (safe-outputs handles the writes)

Then compile your workflow using `gh aw compile`, run it, and verify the issue appeared on this repo.

## Tips

- Run `gh aw --help` to see available commands and flags
- The simplest frontmatter has just `on`, `permissions`, `safe-outputs`, and `engines`
- Your workflow **body is just natural language markdown** — no special syntax, no code blocks required. The AI agent reads your instructions and does the work.
- Start with `engine: copilot` if you're unsure (included free with your GitHub account)
- Common pattern: instructions like "Create a GitHub issue with..." followed by what you want the issue to say
- Don't overthink it — a 3-sentence greeting is more than enough for the AI to work with

## Steps (High Level)

**You'll do these; figure out the exact commands yourself:**

1. Set up your environment (if not already done) using the [devcontainer setup guide](../../docs/getting-started/devcontainer-setup.md)
2. Create `.github/workflows/hello-agent.md` with frontmatter + body
3. Run `gh aw compile` to generate the `.lock.yml`
4. Trigger the workflow manually via the GitHub Actions UI (or use `gh workflow run`)
5. Watch the logs and verify the issue was created
6. Check the Issues tab — your first agent-created issue should be there
